Scanning with Nmap

  • 先確認該機器可以連到網路，VM記得切NAT

這台機器無法透過 ifconfig 或 ip a 知道IP，要透過kali去找

  • ARP的運作原理
    • 在乙太網路上，資料的傳遞必須要有實體位址 (MAC Address)
    • Layer 2 設備會驗證 Frame 的實體位址，不是找它的一律捨棄
    • 設備一開機總不可能就有所有設備的實體位址
    • 需要 ARP 協定來協助取得各個設備的實體位址
    • 利用乙太網路的廣播功能所設計出來的位址解析協定
    • ARP 快取內建立 ARP 表格，查詢的方式來獲得 I P位址 (IP Address) 和實體位址 (MAC Address / Physical Address) 的對應關係
    • Layer 3 的設備都一定會有 ARP Cache
  • 到目標kioptrix右鍵點選Setting > Network Adapter NAT > Advanced
  • 找到目標機器的MAC Address

  • 根據剛kali掃描結果比對目標的IP位置

掃描開放的Port、服務..

  • 使用Nmap

    • 常見參數

      • -A 偵測主機的作業系統與各種服務的版本
      • -O 作業系統資訊
      • -sV 只需要各種服務版本
      • -sA 偵測主機是否啟用防火牆
      • -PN 防火牆保護下的主機
      • -sP 偵測有開機的主機
      • -F 加快掃描速度
      • -T4 比默認速度更快的掃描速度
        • 以更快的速度發送掃描請求和處理回應
        • 可能增加被偵測到的風險
      • -T5 加快掃描速度
        • 最快的掃描速度級別
        • 增加網路負載和被偵測到的風險
        • 可能導致被目標網路的防火牆或入侵偵測系統封鎖或觸發警報
      • -p 掃描連接PORT
      • -p- 掃描所有可能的端口(1~65535範圍內)
      • -sL 查詢網域中所有主機名稱
      • -v 更詳細輸出
      • -Pn 不會進行主機發現，而是直接對指定的目標主機進行連接埠掃描。

      • -sS 指定一種掃描技術，即 TCP SYN 掃描（也稱為半開放掃描或 Stealth 掃描）

        • SYN > SYNACK > ACK
        • 主機回應一個SYN/ACK 套件 > 連接埠開放
        • 主機回應一個RST 套件 > 連接埠是關閉的
        • 目標主機沒有回應 > 連接埠是過濾的或防火牆封鎖了掃描

      • 查看開啟服務及版本並蒐集重要資訊

      • 22/tcp

        • SSH服務的預設端口，用於遠端安全存取
        • 服務版本為OpenSSH 2.9p2，使用的是協定版本1.99
      • 80/tcp
        • HTTP服務的預設端口，用於提供Web服務
        • 主機正在執行一個HTTP伺服器，用於提供網站服務
        • 服務版本為Apache httpd 1.3.20，使用的是Unix系統（可能是Red Hat或Linux）上的Apache HTTP伺服器
        • 安裝了mod_ssl模組（版本為2.8.4）
        • 使用的是OpenSSL函式庫（版本為0.9.6b）來提供安全連線
      • 111/tcp
        • RPC綁定服務的預設端口，用於遠端過程調用（Remote Procedure Call, RPC）
        • 允許客戶端遠端呼叫註冊的RPC程式
      • 139/tcp
        • NetBIOS會話服務的預設端口，用於共用檔案和印表機
        • 工作小組名稱為MYGROUP
      • 443/tcp
        • HTTPS服務的預設端口，用於提供安全的Web服務
        • 服務版本為Apache/1.3.20，使用的是Unix系統（可能是Red Hat或Linux）上的Apache HTTP伺服器
        • 安裝了mod_ssl模組（版本為2.8.4）
        • 使用的是OpenSSL函式庫（版本為0.9.6b）來提供安全連線。
      • 32768/tcp
        • 這是狀態服務的預設端口，通常用於提供服務狀態資訊
        • 該主機正在執行一個服務，它提供有關係統狀態的信息，通常是一個RPC（Remote Procedure Call，遠端過程呼叫）服務，服務編號為100024
