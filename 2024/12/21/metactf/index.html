What is a Buffer Overflow?
When a program is executed, the CPU processes instructions stored in memory. A stack is an area within memory that is used to store variables, manage function calls, and help control the flow of the program.

When a program receives data from a user, it is stored on the stack in a “buffer”, which is a designated chunk of memory reserved for that data.

A buffer overflow occurs when more data is written to a buffer than it can hold. This extra data can overwrite adjacent memory, potentially leading to unexpected behavior or security vulnerabilities.

The RSA algorithm involves the use of public keys for encryption, and private keys for decryption

  1. RSA Setup:

    • Public Key: E (the exponent) and n (the modulus, which is the product of two large prime numbers p and q).
    • Private Key: d (the private exponent), which is computed using the Euler’s Totient Function φ(n) where φ(n) = (p - 1) * (q - 1).

  2. Encryption:

    • The message (the flag in this case) is converted into an integer pt (plain text).
    • The message is encrypted using the formula:

    where E is a fixed public exponent (65537), and n is the product of two large primes p and q.

     $$
ct = pt^E \mod n
$$

  3. Decryption:

    • To decrypt, you need the private exponent d, which is computed using the modular inverse of E modulo φ(n):

       $$

      d = E^{-1} \mod φ(n)
      $$

    • The original message is recovered using:

      $$
      pt = ct^d \mod n
      $$

the flag is encrypted multiple times (1024 times, specifically), with each step involving a new pair of primes p and q. This process produces a new n each time, and the ciphertext (ct) from the previous step is used as the plaintext for the next encryption step.

This chaining of encryptions means the ciphertext from one step is passed on to the next encryption step. If you can reverse this chain, starting from the final ciphertext and moving backward, you can eventually recover the original plaintext (the flag).

Key Observations:

  1. Each n is the product of two primes, p and q, and these are different for each encryption step.

    However, unlike traditional RSA, the primes aren’t chosen randomly each time. They are close enough to each other that certain factoring techniques can work.

  2. Each step only needs to “undo” the encryption from the previous step. This is where the main challenge lies. You have to find the right way to reverse each encryption process.

The Attack Process:

To reverse the encryption chain, we need to:

  1. Extract the Moduli (n): The challenge provides a list of the moduli n used in each encryption step (from output_data.txt). These moduli are the product of two primes p and q.

  2. Factorization of n: In traditional RSA, factoring n into p and q is a computationally hard problem (this is why RSA works as a secure encryption system). However, since the primes p and q are chosen to be close to each other, we can apply Fermat’s Factorization Method to factor n efficiently. This method works well when the two primes are close to each other, as it uses the difference of squares formula:

    $$n=(a+b)(a−b)n = (a + b)(a - b)$$

    where a and b are integers derived from n.

  3. Reversing the Encryption:

    • After factoring n into p and q, we can compute φ(n) (Euler’s Totient Function) and then calculate the private key exponent d using the modular inverse of E modulo φ(n).
    • With d, we can decrypt the ciphertext at each step using:

    This decrypts the ciphertext to recover the plaintext for that step. We then use this as the ciphertext for the next step, continuing until we have decrypted back to the original plaintext.

     $$ct=ctdmod  nct = ct^d \mod n$$

  4. Recovering the Flag: By iterating through all 1024 encryption steps in reverse order, you can eventually recover the original plaintext, which is the flag.

Detailed Steps in Code:

  1. Load the Moduli (n): The moduli used in the RSA encryptions are read from the output_data.txt file, where each n is printed. The code extracts these moduli as integers.
  2. Factor Each Modulus n: Using Fermat’s Factorization method, the code factors each n into its prime components p and q.
  3. Calculate φ(n): For each modulus n, after extracting the primes p and q, the code calculates the totient function φ(n) = (p - 1) * (q - 1).
  4. Calculate the Private Exponent d: The code computes the modular inverse of E modulo φ(n) to get the private exponent d.
  5. Decrypt Each Step: Starting from the final ciphertext, the code iterates backward through the encryptions, applying RSA decryption with the calculated d and the corresponding modulus n at each step.
  6. Recover the Flag: Finally, after completing all decryption steps, the recovered plaintext is converted back to bytes, and the flag is printed.
